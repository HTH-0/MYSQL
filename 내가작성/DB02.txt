계층 데이터 모델

네트워크 데이터 모델

관계 데이터 모델
	속성/attribute : 릴레이션의 열
	투플(tuple) : 릴레이션의 행
	도메인 : 속성이 가질수 있는 모든 값의 집합
	차수 : 하나의 릴레이션에서 속성의 전체 개수(가로/행 길이)
	카디널리티 : 하나의 릴레이션에서 투플의 전체 개수(세로/열 길이)
 
스키마 : 데이터베이스의 구조와 제약조건에 관해 전반적인 명세를 기술한 것
	속성(Att~), 개체(Entity), 관계(Rela~), 제약조건들을 기술한 것
	
테이블 : 2차원 표 형태

인스턴스 > 데이터베이스 > 스키마 > 테이블

수퍼키 : 유일성의 특성을 만족하는 속성/속성들의 집합
유일성 : 키가 갖추어야 하는 기본 특성

기본키 (PK) Primary key 
	여러 후보 키 중에서 기본적으로 사용 할 키. Null이 될 수 없다.

외래키 (FK) Foreign key
	어떤 릴레이션에 소속된 속성, 집합이 다른 릴레이션의 기본키가 되는 키
	다른 릴레이션의 pk를 그대로 참조하는 속성의 집합
	>> 다른 테이블의 pk를 fk라고 한다.
	자기 자신을 참조하는것도 가능 ex) 회원과 추천고객

무결성 제약조건



DDL
	개체를 생성/삭제/변경
	CREATE, DROP, ALTER, USE

DML
	데이터를 검색/수정/삭제
	SELECT, INSERT, UPDATE, DELETE

DCL
	무결성, 보안, 회복, 동시성 제어 역할
	COMMIT, ROLLBACK
