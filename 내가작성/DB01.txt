ER 모델

Entity : 둥근 네모로 표현
>> 테이블 그 자체를 의미

Attribute : 타원으로 표현
개체가 가지고 있는 고유 특성
단일값/ 다중값
	다중값은 타원이 2개 겹친 모양

유도 attribute : 기존 속성의 값에서 유도되는 속성
	실선으로 된 타원으로 표현

Null attribute : 아직 결정되지 않거나 모르는값
공백, 0 과는 다르다.

키 attiribute : 개체를 구성하는 속성(attibute) 중에서도 특별한 역할
속성에 밑줄로 표현


Relationship
개체와 개체가 맺고 있는 연관성

1:1 관계 : 개체A 와 개체B 각각 연결
	ex) 남편 아내
1:N 관계 : A개체에 여러가지 B개체가 연결
	ex) 부서 사원
N:M 관계 : A 여러개 B 여러개 연결
	ex) 고객 책

참여특성
필수참여 : 개체 A,B 관계에서 A의 모든 개체인스턴스가 관계에 반드시 참여 할 때
	실선2개로 이어짐
선택참여 : 반드시 참여하지 않아도 될 때
	실선 1개로 이어짐

관계의 종속성
존재종속 : 개체가 독자적으로 존재 x, 다른 개체의 존재 여부에 의존적
약한 개체 : 다른 개체의 존재 여부에 의존적인 개체
	테두리 선 2개로
오너 개체 : 다른 개체의 존재 여부를 결정하는 개체
	테두리 선 2개로

erd 관계선
| : 1개
까마귀 : 여러개
O : 0개
ex) a O + 까마귀 b : 하나의 a는 0또는 여러 개의 b로 구성
